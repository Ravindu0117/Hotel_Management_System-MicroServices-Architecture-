🏥 Hospital Management Microservices System

This project is a Hospital Management System developed using microservices architecture to demonstrate how real-world applications can be designed as independent, scalable services. The system is divided into multiple microservices, including Patient Service, Doctor Service, Appointment Service, and Medical Record Service, where each service handles a specific functionality and runs independently.

An API Gateway is implemented to provide a single entry point for all client requests, routing them to the appropriate microservice and simplifying communication between the frontend and backend services. Each microservice is built using Node.js and Express, and APIs are documented using Swagger for easy testing and integration.

This project highlights key concepts such as service independence, scalability, maintainability, and API management, providing hands-on experience with modern backend development practices.
